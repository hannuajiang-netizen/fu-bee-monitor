# 🦞 fu之小蜜蜂监控后台

一个基于 React + Flask 的数据分析 Web 后台，用于监控校园认证用户 DAU、大盘 DAU 和社团活动拉新数据。

## 功能特性

### 任务一：校园认证用户DAU监控
- 上传 3 个 Excel 文件（用户基本情况、内容生产情况、累计单校情况）
- 生成近 14 天 × 25 列汇总表
- 自动生成周报结论文本

### 任务二：大盘DAU监控
- 数据位置转换（本周数据 → 下周上周数据格式）
- 新数据填入模板，一键生成周报话术
- 与任务三联动，自动获取 TOP 渠道数据

### 任务三：社团活动拉新监控
- 上传拉新数据报表 Excel
- 渠道复选框管理（支持全选/取消全选/新增/删除）
- 自动发现新渠道
- 生成近一周拉新 TOP10 渠道
- 勾选渠道整体汇总数据（加权平均留存率）
- 各渠道明细表

## 技术栈

- **前端**: React 18 + Ant Design 5 + Vite
- **后端**: Python Flask + Pandas + openpyxl
- **数据存储**: JSON 文件（渠道配置、TOP渠道缓存）

## 启动方式

### 1. 启动后端

```bash
cd backend

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python app.py
```

后端服务将启动在 `http://localhost:5000`

### 2. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端服务将启动在 `http://localhost:3000`

### 3. 访问应用

打开浏览器访问 `http://localhost:3000`

## 使用说明

### 任务一：校园认证用户DAU监控

1. 上传 3 个 Excel 文件
2. 点击「开始分析」
3. 查看 14 天汇总表和周报文本
4. 点击「复制」按钮复制结果

### 任务二：大盘DAU监控

**功能区 1 - 数据位置转换：**
1. 在左侧粘贴原始数据（如：日均DAU 66499（上周62522）...）
2. 点击「转换」按钮
3. 右侧显示转换后的格式

**功能区 2 - 生成周报话术：**
1. 在输入框粘贴本周数据（Tab 或空格分隔）
2. 点击「一键填入」
3. 确认各字段数值
4. 点击「生成周报话术」
5. 复制生成的两份输出

### 任务三：社团活动拉新监控

1. 上传拉新数据报表 Excel
2. 选择时间范围
3. 勾选需要统计的渠道（支持全选/取消全选）
4. 点击「开始分析」
5. 查看 TOP10 渠道、整体汇总和各渠道明细

## 任务间联动

任务二的「本周主要由______带来新增」会自动从任务三获取数据：
- 先在任务三执行一次分析
- 任务二会自动获取任务三最近一次分析的 TOP 渠道
- 如果任务三尚未执行，会显示提示信息

## 数据格式要求

### 任务一 - 用户基本情况表
应包含字段：日期、活跃用户数、人均停留时长(分钟)、互动次数、互动人数、互动人数占比、分享人数、关注人数、次日留存率、3留率、7留率、14留率、30日留率、活跃在群用户数、群活跃用户数、群发言用户数、活跃在蜂巢用户数、蜂巢活跃用户数

### 任务一 - 内容生产情况表
应包含字段：日期、当日发布笔记量、当日发布笔记用户数、累计发布笔记量、累计发布笔记用户数

### 任务三 - 拉新数据报表
应包含字段：拉新日期、拉新归属、拉新人数、次日留存、次日留存率、3日留存、3日留存率、7日留存、7日留存率、14日留存、14日留存率、30日留存、30日留存率、人均使用时长

## 注意事项

- 所有数字去除千位分隔符
- 百分比格式为 XX.XX%
- 表格复制为 Tab 分隔格式，可直接粘贴到 Excel/飞书
- 上传的文件会自动保存到 `./uploads/` 目录
- 渠道配置和 TOP 渠道结果会持久化保存
